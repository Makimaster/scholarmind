# 提示词：多查询扩展 (multi_query)

- **用途**：一个问题生成多种同义/不同角度的问法，扩大召回覆盖，避免单一表达漏召回。
- **输入**：`{question}`、`{n}`（生成几条，默认 3）
- **输出**：JSON 字符串数组。

## Prompt
```
为下面的问题生成 {n} 个语义等价但表达不同的检索查询，覆盖不同角度和术语。
保持专业、简洁，不要重复原句。

问题：{question}

只输出 JSON 数组：["...", "...", "..."]
```

## 优化建议
- 学术场景可让其中一条偏"术语化"、一条偏"通俗化"。
- 与跨语言结合时，可要求生成中英混合的多条。
